Intro

An Object is a container-like data structure. It can be used to group related data or functionality together. Arrays, variables, functions and even other objects can be stored inside an object.

Illustration

An object is like a box labeled kitchen. Inside are things related to the kitchen. 

var kitchen = { 
    stove:"Whirlpool",
    drawer:["forks","knives","spoons"],
    cookDinner:function(){
        console.log("Dinners Ready!");
    },
    refrigerator:{ //object in object
        shelves:2,
        contents:['oranges','apples','bacon'],
        clean:function(){
            console.log("All Clean!");
        }
    },
};
Create Object
var myObj = {}; //empty object
var myObj = {color:"blue"};

Data inside objects are stored in key:value pairs. The key is the member name or identifier used to access the value associated with it. Object members that are not functions are called properties. Object members that are functions are called methods. Member values can be any data type.

Create Object Members
var myObj = {fName:"Tim",
            lName:"Jones"};
//OR
myObj["fName"] = "Tim"; //Bracket Notation
myObj.lname = "Jones"; //Dot Notation
myObj["pName"]=function(){
    return myObj.fName + " " + myObj.lName;
};

Object properties and methods can be added using the Dot Notation or the Bracket notation. They can also be created when the object is created

var myObj = {
    fName:"Tim",
    lName:"Jones",
    pName: function(){
        return this.fName + ' ' +  this.lName,
}};
Access Members
myObj["fName"] -> "Tim" //Bracket Notation >
myObj.lName -> "Jones" //Dot Notation 
myObj.pName() -> "Tim Jones" //access method>

Data inside objects can be accessed using the dot notation or the bracket notation. Methods are also accessed by dot or bracket notation, but otherwise behave like normal functions. Data has to be accessed using the object's name followed by the property/method name. If a property is nested in more than 1 object,then all "parent" objects must also be written to access the data.

var kitchen = { 
    stove:"Whirlpool",
    drawer:["forks","knives","spoons"],
    cookDinner:function(){
        console.log("Dinners Ready!");
    },
    refrigerator:{ //object in object
        shelves:2,
        contents:['oranges','apples','bacon'],
        clean:function(){
            console.log("All Clean!");
        }
    },
};
kitchen.refrigerator.clean(); -> "All Clean!"
Change Member Values
myObj["fName"] = "Mike" -> Mike
myObj.lName = "Smith" -> Smith

Member values can be changed simply by accessing them using dot or bracket notation, and then changing the value.

This
var greeting = {
    name:"Jim",
    sayHi:function(){
	var name = "Liz";
	console.log(name);  -> "Liz"
	console.log("Hi " + this.name); -> "Hi Jim"
    }};
greeting.sayHi()

This refers to the current object where 'this' is written. "this" object is 'greeting' and has a variable named name. Without 'this', name refers to the variable defined inside the function. this.name == greeting.name

Possible Uses

Having a lot of variables can be hard to maintain and scale.

var user1fname = "Tim"
var user1email = "Tim@email.com"
var user1profilepic = "Tim.png"
var user2fname = "Liz"
var user2email = "Liz@email.com"
var user2profilepic = "Liz.png"

Objects make it easier. 

var Tim = {
    fname:"Tim",
    email:"Tim@email.com,
    pic:"Tim.png"};
var Liz = {
    fname:"Liz",
    email:"Liz@email.com,
    pic:"Liz.png"};

Now all the information about the users are stored together for easier use.
