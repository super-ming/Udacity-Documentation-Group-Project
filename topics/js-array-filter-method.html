Intro

One of the useful built-in methods for working with arrays is the filter() method. By using the filter() method, we can take an array and filter it down into a new array containing only the elements that satisfy a given criteria.

The criteria to obtain the desired array elements from an array is in the form of a function that is passed into the filter() method, this is called a predicate function. The predicate function that is passed into filter() will be ran on each array element in the array that the method is called on. As each array element passes through this function, a boolean true is returned for array elements that have been returned as true from the function body, only array elements returning true will be included in the resulting array.

Because the filter() method returns a new array rather than modifying an array, a variable is created to hold the new array that contains the filtered array elements.

Without the filter() method, programmers would have to use a for.. loop to achieve the same functionality that the filter() method provides. However, using the filter() method is regarded as a better practice as it makes the source code more readable and more concise.

The filter() method creates a new array with all elements that pass the test implemented by the provided function.

Taken from the Mozilla Developer Network page on Array.prototype.filter()

The filter() method was introduced in ECMAScript 5, so a polyfill or a JavaScript utility library will be necessary for using filter() in older browsers.

Illustration

To better grasp the concept of the filter() method, we can imagine a gold miner wanting to filter out gold and gems from a collection of mined rock. In a code context, this concept can be illustrated by using an array for the collection of mined rocks, and the array elements as being either rock or more valuable minerals such as gold or gems.

                  
                // our mined rock consists of rocks and more valuable materials
                const minedRocks = [
                  'rock',
                  'gem',
                  'rock',
                  'rock',
                  'gold',
                  'rock',
                  'rock',
                  'rock',
                  'gem',
                ];

Our miner uses a method called gold panning in order to obtain the valuable materials from the collection of rocks. Gold panning is a process of agitating some raw material in a pan with water so that the gold will sink to the bottom of the pan to be recovered while the rest of the materials will spill out of the pan and be discarded. In a code context, since panning is an action we can use a function called goldPanning, and pass in each piece of raw material as a parameter to be tested, goldPanning(material). The test itself, found in the function body, can be in the form of a simple if statement, where the condition will return true if a piece of rock is either gold or a gem.

When using the filter() method and designing the callback function to be passed into it, we need to be aware that only the array elements that return true from the callback function will be included in the resulting array.

                  
                // technically, this is our callback function
                function goldPanning(rock) {
                  if (rock === 'gem' || rock === 'gold') {
                    return true;
                  }
                };

Moving on, since our miner plans on selling the gold he finds, we need to facilitate a way to hold onto the pieces of gold that were found from the panning process. In code terms, we first need to distinguish between when a built-in array method modifies an array from when a built-in array method results in a new array. Luckily for the miner, the filter() method does the latter, and the method results in a new array so the miner can easily store the valuable materials he finds. In our code, we would simply create a new variable called valuableMinerals that is used for storing the valuable minerals that were returned from running our goldPanning function.

                  
                const valuableMinerals = minedRocks.filter(goldPanning);

To finish our illustration, let's see how many pieces of valuable materials were found by using console.log and the built-in .length() property for arrays. To see the actual resulting array, with the valuable minerals that were obtained from the gold panning process, just pass in the variable name used.

                  
                console.log(valuableMinerals.length); // returns 3

                console.log(valuableMinerals); // returns ["gem", "gold", "gem"]

The Call

Since the filter() method is a built-in array method, the method can act upon an array by simply appending .filter(), and then providing a callback function. Since the filter() method returns an array, a variable is used to save the new array containing the filtered array elements.

                  
                var filteredArray = arrayToFilter.filter(callbackFunction(currentValue, index, arr), thisValue);

In the gold miner illustration, a callback function named goldPanning() was passed into filter() as a parameter, yet we could have simply provided that function as an anonymous function instead.

                  
                // old way
                // function goldPanning(el) {
                //   if (el === 'gem' || el === 'gold') {
                //     return true;
                //   }
                // };
                //
                // const valuableMinerals = minedRocks.filter(goldPanning);


                // more concise way, using an anonymous function
                const valuableMinerals = minedRocks.filter(function (material) {
                  if (material === 'gem' || material === 'gold') {
                    return true;
                  }
                });

Possible Uses

The filter() method can be applied for a myriad of uses. Generally, whenever a developer whats to find certain array elements that are contained withing an array, the filter() method can be used as a way to obtain those specific array elements and return them in a new array.

Perhaps the coolest thing to note about the filter() method, is that it returns a new array instead of modifying an array, which allows it to be chainable with other array methods that return a new array. Allowing the filter() method to be combined or chained with other array methods such as sort() and map() which are built-in methods that also return a new array instead of modifying it.

Chaining methods allows a developer even greater control when obtaining an array with only specific items, it would look similar to this example in a JavaScript file:

  
// the "…" is where anonymous functions would likely be added
myArray.filter(…).map(…).sort(…); // returns a new array

Some uses for using filter() may include:

  • Finding persons who are older, younger or within a range of ages.
  • Finding certain array elements in an array that meet a certain criteria.
  • Finding certain persons in an array who have a specific name.
  • When used with the array length() property, you can determine how many array elements have been filtered into the resulting array.
  • Determining if a certain object property is the correct data type, string length, ect.
  • After you receive some JSON data from an Ajax call, you can filter out only the disired data from the JSON.

Also note, that the filter() method does not run on array elements that do not have a value.
